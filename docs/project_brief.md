# Implementation Brief for the Portfolio Website

## 1. Overall creative direction summary

This portfolio should feel like a premium, design led digital experience with strong authorship.

It is not a generic developer portfolio. It should feel like the intersection of a high end editorial website, a cinematic product experience, and a creative developer portfolio.

The site should communicate that Zaw Htike Aung is:

1. A premium frontend developer
2. A creative developer
3. A design aware builder
4. Someone who understands both visual design and engineering
5. Someone who can create digital work with atmosphere, structure, and polish

The overall emotional tone should be:

1. Calm
2. Refined
3. Slightly mysterious
4. Editorial
5. Modern
6. Quietly confident
7. Tasteful
8. Minimal but expressive

The website should feel expensive through restraint, composition, typography, spacing, and motion control.

It should not feel loud, trendy, playful, flashy, or startup templated.

The experience should feel authored from beginning to end.

## 3. Design system summary

### Typography

Font pairing is locked.

**Display font**

PP Editorial New

**Use for**

1. Hero headline
2. Major section titles
3. Signature statements
4. Occasional identity moment

**Sans font**

Neue Haas Grotesk

**Use for**

1. Subheadlines
2. Body copy
3. Navigation
4. Buttons
5. Project descriptions
6. Labels
7. Metadata
8. Footer

**Typography rules**

PP Editorial New should create emotional peaks. Neue Haas Grotesk should carry structure and readability. Most text should be left aligned. Sentence case should dominate. All caps should be used only for small labels. Paragraphs should stay short and visually composed.

**Desktop scale guidance**

1. Hero display: 84px to 112px
2. Major section title: 48px to 64px
3. Subheadline: 20px to 24px
4. Body large: 18px to 20px
5. Body standard: 16px
6. Labels and metadata: 12px to 14px

**Mobile scale guidance**

1. Hero display: 44px to 60px
2. Major section title: 32px to 40px
3. Subheadline: 18px to 20px
4. Body: 15px to 18px

**Color behavior rules**

1. No pure black
2. No pure white
3. Most of the site should live in dark neutrals
4. Accent should appear sparingly
5. Warm support tone should be extremely rare

### Material system

The material direction is architectural restraint with cinematic depth.

**Surface hierarchy**

1. Level 0 canvas
2. Level 1 lifted surface
3. Level 2 framed inset or denser object surface

**Corner radius system**

1. Large: 16px
2. Medium: 10px
3. Small: 6px

**Shadow system**

Very soft, dark, diffuse. Depth should be felt before it is seen.

**Blur usage**

Minimal and atmospheric only. Use lightly on floating nav or atmospheric layers. Do not build the visual identity around blur.

**Texture**

Very subtle grain only. Almost invisible. Never strong enough to compete with text.

**Border behavior**

Mostly 1px borders. Fine structural framing. No thick ornamental outlines.

### Layout and spacing

The grid should feel editorial and architectural.

**Principles**

1. Large negative space
2. Strong outer margins
3. Asymmetrical composition when useful
4. One clear focal point per viewport
5. Section pacing should vary in visual pressure

Content should feel placed, not stacked.

### Buttons

Buttons should feel like refined control surfaces.

**Primary button**

Light surface with dark text

**Secondary button**

Transparent or dark surface with fine border

**Button rules**

1. No glossy effects
2. No oversized playful pill shape
3. No aggressive shadows
4. Hover should feel tactile and subtle

## 4. Motion direction summary

Motion should feel like camera intelligence applied to interface.

The motion character should be:

1. Soft
2. Weighted
3. Precise
4. Cinematic
5. Non flashy
6. Controlled
7. Calm

### Core motion behavior

Use mostly:

1. Fade
2. Small rise
3. Mask reveal
4. Small scale normalization
5. Soft parallax
6. Tonal transition

### Timing tokens

1. Micro interaction: 160ms
2. UI transition: 240ms
3. Content reveal: 560ms
4. Image reveal: 780ms
5. Hero reveal: 920ms
6. Signature reveal: 1100ms

### Easing tokens

1. Primary easing: `cubic-bezier(0.22, 1, 0.36, 1)`
2. Secondary easing: `cubic-bezier(0.4, 0, 0.2, 1)`
3. Cinematic easing: `cubic-bezier(0.16, 1, 0.3, 1)`

### Reveal rules

**Small text reveal**

1. Opacity from 0 to 1
2. Y offset around 12px to 18px

**Large title reveal**

1. Mask or clip reveal plus soft rise
2. Y offset around 20px to 28px

**Image reveal**

1. Opacity plus larger rise
2. Optional very slight scale from 1.015 to 1
3. Blur only in rare major moments

### Hover rules

**Buttons**

1. Slight tonal shift
2. Tiny lift only if needed
3. No bounce

**Links**

1. Subtle brightness shift
2. Quiet underline or line detail

**Projects**

1. Lift by 2px to 4px
2. Image scale no more than 1.015
3. Border shift minimal
4. Caption opacity may increase

### Scroll behavior

Section reveals should begin just before the section fully enters view. Parallax should be used only in a few meaningful places. Pinned sections should be rare and short. Stillness is part of the experience.

### Reduced motion

The site must remain elegant without heavy motion. Remove parallax and long atmospheric motion for reduced motion users. Keep structure, contrast, and typography strong enough to carry the experience without kinetic dependency.

## 6. Key visual rules

### 1. One dominant focal point per viewport

Each section should have one clear visual priority.

### 2. Typography is a main event

Type should not just hold information. It should create mood, hierarchy, and identity.

### 3. Negative space must stay active

Do not fill space because it feels empty. Empty space is part of the premium effect.

### 4. Project presentation must be art directed

Project visuals should feel presented, not dropped in. Use consistent image framing and tonal control.

### 5. Surfaces should feel framed, not card based

Avoid a generic product card feeling. Use planes, mounts, and fine edges instead of obvious UI furniture.

### 6. Motion should reveal, not decorate

All movement should support focus, atmosphere, pacing, or tactility.

### 7. Accent color is a precision tool

Use the accent sparingly and intentionally.

### 8. Contrast should be softened

Avoid harsh black and white contrast. Use tonal layering and restrained material depth.

### 9. Section rhythm must vary

Some sections should feel suspended and spacious. Others should feel more grounded or visual. Do not make every section feel equally dense.

### 10. The ending must leave an afterimage

The final third of the page should still feel authored, not like the site is winding down.

## 7. What must be avoided

Avoid anything that pushes the site toward generic premium portfolio territory.

### Visual things to avoid

1. Pure black backgrounds
2. Pure white text
3. Heavy gradients
4. Neon accents
5. Glossy surfaces
6. Visible glassmorphism
7. Thick borders
8. Overly round startup UI
9. Too many floating objects
10. Too many cards
11. Visible noise textures
12. Unstyled screenshots
13. Generic laptop mockups used lazily

### Motion things to avoid

1. Bounce
2. Overshoot
3. Springy default motion everywhere
4. Loud stagger on every element
5. Constant floating animations
6. Long pinned storytelling sequences
7. Big travel distances
8. Parallax on everything
9. Motion that draws attention to the library instead of the experience

### Content things to avoid

1. Tech badge clouds
2. Long skill lists in the hero
3. Generic services section
4. Too many homepage projects
5. Resume style timeline on the homepage
6. Salesy CTA language
7. Portfolio clichés about passion or innovation
8. Fake luxury wording
9. Explaining too much too early

### Structural things to avoid

1. Hero that tries to say everything at once
2. Homepage that becomes a checklist of sections
3. Too many visual ideas in one viewport
4. Project showcase that looks weaker than the homepage shell
5. Mobile layout that collapses into ordinary stacked cards
6. Footer that feels like an afterthought

## 8. Technical implementation direction for Next.js, Tailwind CSS, GSAP, and React Three Fiber

### Next.js

Use App Router if possible. Keep structure modular and section based. Make each homepage section a dedicated component. Keep layout and motion logic clean and isolated.

**Suggested component structure**

1. HeroSection
2. ThesisSection
3. WorkIntroSection
4. FeaturedProjectSection
5. ProcessSection
6. AboutSection
7. ValueSection
8. SignatureSection
9. ContactSection

### Tailwind CSS

Define the full system in theme tokens.

Create tokens for:

1. Colors
2. Font families
3. Font sizes
4. Spacing
5. Radius
6. Shadows
7. Transition durations
8. Easing
9. Z index layers

Keep utility use consistent with the design system. Avoid ad hoc styling that breaks the visual discipline.

### GSAP

Use GSAP for:

1. Hero reveal
2. Section reveal orchestration
3. Scroll linked depth
4. Signature moment transitions
5. Rare pinned moments if needed

Use GSAP with restraint. Keep animations low amplitude and tied to layout and storytelling.

### React Three Fiber

Use R3F only if it adds real atmosphere or spatial identity.

**Good uses**

1. Subtle hero depth field
2. Minimal abstract spatial object
3. Soft environmental motion layer
4. Light driven scene support

**Bad uses**

1. Showpiece 3D for its own sake
2. Interactive object that competes with typography
3. Heavy canvas performance cost
4. Obvious creative developer spectacle

If R3F is used, it should behave like ambient set design, not the star of the page.
